# セキュリティガイドライン

## コミット前の必須セキュリティチェック

すべてのコミット前に確認すること:
- [ ] シークレットがハードコードされていない（APIキー・パスワード・トークン）
- [ ] ユーザー入力がすべて検証されている
- [ ] SQLインジェクション対策（パラメータ化クエリを使用）
- [ ] XSS対策（HTMLが適切にサニタイズされている）
- [ ] CSRF保護が有効になっている
- [ ] 認証・認可が適切に実装されている
- [ ] すべてのエンドポイントにレートリミットが設定されている
- [ ] エラーメッセージが機密情報を漏洩しない

## シークレット管理

- シークレットをソースコードに絶対にハードコードしないこと
- 環境変数またはシークレットマネージャーを使用すること
- アプリ起動時に必要なシークレットが存在するか検証すること
- 漏洩した可能性のあるシークレットは必ずローテーションすること

## セキュリティ対応プロトコル

セキュリティ上の問題が見つかった場合:
1. 作業を**即座に停止**する
2. **code-reviewer** agent を使ってセキュリティ観点からレビューを依頼する
3. CRITICAL な問題を先に修正してから作業を再開する
4. 漏洩した可能性のあるシークレットをローテーションする
5. コードベース全体で類似の問題がないか確認する
