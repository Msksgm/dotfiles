# Agent オーケストレーション

## 利用可能な Agent

`~/.claude/agents/` に配置されている agent:

| Agent | 目的 | 使うタイミング |
|-------|------|---------------|
| **architect** | DDD ベースのソフトウェアアーキテクチャ設計 | 新機能の設計・集約境界の決定・レイヤー構造の検討 |
| **code-reviewer** | コード品質・セキュリティ・保守性の検証 | コードの変更後 |
| **pm-reviewer** | プロダクトマネージャー視点での仕様レビュー | プラン作成後・仕様レビュー時 |
| **qa-reviewer** | テスト網羅性の検証・不足テストケースの特定 | 実装完了後・フェーズ完了時 |

## Agent の即座の使用

以下の状況ではユーザーのプロンプトを待たずに積極的に使うこと:
1. 複雑な機能リクエスト → **architect** agent
2. コードを書いた・変更した直後 → **code-reviewer** agent
3. バグ修正や新機能実装後 → **qa-reviewer** agent
4. アーキテクチャ上の意思決定が必要 → **architect** agent
5. プランや仕様の確認 → **pm-reviewer** agent

## 並列タスク実行

独立した操作には常に並列 agent 実行を使うこと:

```markdown
# GOOD: 並列実行
3つの agent を並列で起動する:
1. Agent 1: 認証モジュールのセキュリティ分析
2. Agent 2: キャッシュシステムのパフォーマンスレビュー
3. Agent 3: ユーティリティのコード品質確認

# BAD: 不要な逐次実行
まず Agent 1、次に Agent 2、最後に Agent 3
```

## 多視点分析

複雑な問題には、異なる役割の agent を分けて使うこと:
- **code-reviewer**: コードの正確性・セキュリティ観点
- **architect**: アーキテクチャ・設計観点
- **qa-reviewer**: テスト可能性・品質保証観点
- **pm-reviewer**: 要件・ユーザー価値観点
